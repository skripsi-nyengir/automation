🏗️ 3 Pendekatan Arsitektur

Berdasarkan kebutuhan: tugas kuliah, slider control, STA mode WiFi

⭐ A

ESP32 All-in-One (Recommended)

Cara kerja: ESP32 jalanin 3 peran sekaligus — web server (hosting HTML/CSS/JS), WebSocket server, dan PWM controller untuk servo.

Komunikasi: Browser → WebSocket → ESP32 → PWM → Servo

Pro: Single device, zero setup (buka browser → langsung kontrol), latency rendah

Kontra: UI terbatas karena memory ESP32, musti flashing ulang kalau mau ubah web

B

Two-Tier: Web Terpisah + ESP32 API

Cara kerja: ESP32 hanya jalanin REST API server. Web page dihosting terpisah (laptop/PC).

Komunikasi: Browser → Laptop Web Server → HTTP REST → ESP32 → PWM → Servo

Pro: UI kaya & fleksibel (React/Vue bebas), easy to debug

Kontra: Setup ribet (2 server), latency lebih tinggi, dependensi lebih banyak

C

ESP32 Embedded + REST

Cara kerja: ESP32 host web page via HTTP, setiap slider digerakkan kirim HTTP request. No WebSocket.

Komunikasi: Browser → HTTP POST → ESP32 → PWM → Servo

Pro: Paling sederhana secara kode, gampang debug via browser

Kontra: Delay karena HTTP handshake tiap gerakan, kurang halus untuk real-time control
