🔄 Data Flow & Error Handling

Flow Lengkap

1. User geser slider "Base" ke 45°
2. app.js: client.publish('robot/servo/base', '45')
3. MQTT Broker terima → forward ke ESP32
4. ESP32 callback: servoBase.write(45)
5. Servo bergerak ke 45° ✅

⏱️ Latency total: ~10-50ms (dalam jaringan lokal)

Error Handling

🛑 WiFi Putus
ESP32 auto-retry connect tiap 5 detik. Servo tetap di posisi terakhir.
🔴 MQTT Disconnect
Browser: status merah + slider disable. ESP32: reconnect otomatis.
⚠️ Nilai Invalid
ESP32 validasi 0-180. Nilai di luar range diabaikan + log error.
🆘 Emergency Stop
Tombol kirim 'STOP' ke topic robot/emergency. ESP32 nonaktifkan semua servo.
🐳 Docker Down
docker compose restart. ESP32 auto-reconnect saat broker kembali.
